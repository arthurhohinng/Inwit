# Sprint 3 Planning

## Sprint goal:
 - Finish user stories: __IN-2__, __IN-8__, __IN-28__, __IN-66__, __IN-67__, __IN-68__, __IN-80__ to focus on vendor view experience.


---

## Tasks Breakdown:

Story: __IN-2__ - As an admin, I want to be able to manage information on containers, so that I can keep inventory of resources

Tasks:
 - View containers in database through admin view
 - Add and edit containers to database through admin view

---


Story: __IN-8__ - As a user, I want to be able to receive and manage impact points, so that for returning containers rewards me

Tasks:
 - Show how many impact points the user has received for each returned container
 - For each pending container, provide an option to show the location of where to return it
 - Show 10 containers per page with the option to go to the next page

---


Story: __IN-28__ - As a restaurant, I want to be able to mark order as confirmed, getting ready, scanned container, order ready to pickup,order completed, order declined

Tasks:
 - Add correct order status options
 - During preparing order status, add option to scan container to associate it with order

---


Story: __IN-66__ - As a restaurant, I want to be able to assign containers to each order item, so that I can track the containers that are given to customers

Tasks:
 - Add function to scan a container
 - Spike: Research how to scan containers, given Inwit's container NFC technology

---


Story: __IN-67__ - As a restaurant, I want to be able to accept a returned container by scanning it, so that I can update the container status by asking me about its condition (broken/lost/good to reuse)

Tasks:
 - Add function to accept container
 - Add button to return a container
 - Spike: Research how to scan containers, given Inwit's container NFC technology

---

Story: __IN-68__ - As a restaurant, I want to edit order status through buttons instead of a dropdown, so that I save time

Tasks:
 - Add buttons for restaurant order status

---


Story: __IN-80__ - As a restaurant, I want to be able to view containers associated with all orders, so that I can easily see what containers of mine are out

Tasks:
 - Add function to display containers for the order

---
